# ARPG Plan

## Ziel

Eine actionlastige RPG-Mod mit klarer Progression, staerkerem Loot-Fokus und wiederholbaren Herausforderungen.

## Erste Systeme

1. Combat Core: Schadenstypen, kritische Treffer, Ausweichen, Trefferfeedback.
2. Player Progression: Level, Erfahrung, Attribute, Skillpunkte.
3. Loot: Seltenheiten, zufaellige Affixe, Item-Power, Drops aus Gegnern.
4. Gegner: Elite-Modifikatoren, Boss-Prototypen, Skalierung nach Gebiet.
5. Quests: einfache Objectives, Belohnungen, spaeter Questketten.
6. UI: Charakterwerte, Loot-Vergleich, Skill-Ansicht.

## Arbeitsweise

- Kleine, spielbare Iterationen bauen.
- Erst Server-Logik stabilisieren, danach Client-UI und Effekte.
- Jedes neue System mit einem minimalen Test-/Debug-Weg ins Spiel bringen.
- Datengetriebene Inhalte bevorzugen, sobald die Kernmechanik steht.

## Naechster sinnvoller Schritt

Ein erstes Attribute-/Progression-System implementieren und im Spiel sichtbar machen, z. B. mit einem Debug-Command oder einfachem HUD.
